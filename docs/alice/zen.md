# The Zen of Alice

The Zen of Alice captures the guiding principles for Alice language design. These are not suggestions or preferences - they are the organic law that every language feature, syntax decision, and design choice must follow.

When in doubt, refer back to these principles. If a feature contradicts them, the feature is wrong, not the principles.

```text
ECS is the language, not a pattern.
Data and behavior are separate.
Composition over inheritance. Always.
Flat hierarchies beat deep nesting.
Minimal boilerplate is a feature.
Readability emerges from simplicity.
Performance comes from structure, not optimization.
Constraints enable freedom.
There should be one obvious way to do it.
Mutations are declared, not discovered.
The mental model stays small.
```
